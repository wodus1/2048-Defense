# 2048 Defense
**2048 퍼즐과 디펜스를 결합한 하이브리드 캐주얼 게임입니다**  
2048 타일을 합쳐 경험치를 모으고, 레벨업 시 랜덤 강화 효과를 선택하여 강해지고 아이템을 사용하여 몰려오는 몬스터 웨이브를 막아냅니다

## 프로젝트 개요
단순 기능 구현에 그치지 않고 **확장 가능한 시스템 구조**, **데이터 중심 설계**,  
**로컬 우선 저장 + 서버 동기화 흐름**, **반복 생성 비용과 GC Alloc을 줄이기 위한 최적화**를 함께 고려해 개발했습니다.

## 핵심 구현 포인트
- `GameManager`에서 2048 보드 이동, 합체, 스폰, 입력 처리, 게임 흐름을 직접 관리
- `ISubSystem` + `SubSystemsManager` 구조로 전투/레벨/강화/스탯 시스템 모듈화
- `ScriptableObject` 기반 `Effect` / `ItemEffect` 설계로 데이터 중심 확장
- `SaveManager` / `SaveService` 분리로 저장 정책과 저장 처리 책임 분리
- 로컬 우선 저장, Pending 백업, revision 비교 기반 서버 저장 구조를 설계
- `ObjectPool<T>`, tween 재사용, 문자열 갱신 최적화로 런타임 비용 절감

## ▶ 주요 기능
- **2048 퍼즐 + 디펜스 결합**  
  2048 보드에서 타일을 합쳐 높은 숫자를 만들수록 더 많은 경험치를 획득합니다

- **웨이브 후 ‘보드 유지’ 시스템**  
  웨이브 종료 시 **가장 큰 타일만 남기고 보드를 초기화**하여 이어서 2048을 진행할 수 있습니다

- **자동 공격 시스템**  
  플레이어는 퍼즐에 집중하고, 투사체는 자동으로 가장 가까운 몬스터를 공격합니다

- **Roguelike 강화 선택**
  레벨업 시 *랜덤 강화 중 1개*를 선택해 강화합니다
  
- **아이템 시스템**  
  소모성 아이템을 장착하고 사용하여 전투에 영향을 줄 수 있습니다

- **웨이브 기반 몬스터 시스템**  
  웨이브가 진행될수록 몬스터 체력·스폰 속도가 증가하며 난이도가 상승합니다

- **UI/연출(DOTween)**  
  타일 팝 애니메이션, 체력바 흔들림, 강화 선택, 브레이크 타임 등 다양한 UI 연출

- **데이터 저장 및 오프라인 대응**  
  데이터 변경 사항은 우선 로컬에 반영되며, 서버 저장 실패 시 Pending 데이터를 유지합니다
  이후 로그인 시점에 서버 저장을 재시도하도록 구성했습니다

## ▶ 게임 플레이

### ▸ 조작 방법
- **2048 타일 이동**
  - 마우스 드래그(또는 터치)를 상/하/좌/우로 스와이프하여 해당 방향으로 타일을 이동·합체할 수 있습니다
- **스탯 확인**
  - 게임 내 **스탯 버튼**을 눌러 현재 공격력, 공격 속도, 투사체 속도를 확인할 수 있습니다
- **아이템 사용**
  - 게임 내 아이템 아이콘을 클릭하면 아이템이 사용됩니다
- **게임 오버 조건**
  - 몬스터가 체력바(HpBar)에 닿으면 주기적으로 데미지를 입습니다
  - 체력이 0이 되면 GameOver UI가 표시되어 재시작 또는 로비로 이동할 수 있습니다


### ▸ 진행 흐름
**2048 퍼즐 플레이**  
- 타일을 합치면 숫자에 비례한 경험치 획득

**레벨업 발생**  
- 경험치가 일정량을 넘으면 레벨업

**강화 선택(Upgrade)**  
- 랜덤 2개의 효과 중 1개 선택  
- 공격력/공격속도/투사체 속도 등 스탯 강화

**자동 전투 진행**  
- 강화된 스탯으로 투사체가 자동 공격  
- 몬스터는 아래로 이동하며 기지 HP를 공격

**아이템 사용**  
- 소모성 아이템을 사용하여 몬스터 공격, 버프 등 특수 효과를 발생 

**웨이브 종료 → 휴식 → 다음 웨이브 시작**
- 웨이브 종료 시 **가장 큰 타일만 남기고 보드를 초기화**하여 2048을 이어서 진행할 수 있습니다

## ▶ 시스템 아키텍처
이 프로젝트는 2048 퍼즐 로직과 디펜스 전투 로직을 분리하고, 인게임 핵심 기능을 `ISubSystem` 기반으로 모듈화해 구성했습니다.
- `GameManager`는 퍼즐 보드와 전체 게임 흐름을 조정하는 중심 관리자 역할을 담당합니다
- 전투, 레벨, 강화, 스탯 등 인게임 핵심 기능은 `ISubSystem` 단위로 분리했습니다
- 공통 초기화와 시스템 참조 관리는 `SubSystemsManager`가 담당합니다
- 강화 효과와 아이템은 `ScriptableObject` 기반으로 구성해 코드 수정 없이 데이터 중심 확장이 가능하도록 설계했습니다
- 데이터 저장은 `SaveManager + SaveService`로 분리해 저장 정책과 저장 처리 책임을 나눴습니다
  
## ▶ 주요 시스템 개요

### GameManager (2048 퍼즐 + 게임 흐름 제어)
- 4×4 2048 보드의 **타일 이동·합체·스폰 로직**을 관리합니다
- `Unity Input System`을 사용해 드래그 방향을 계산하고 해당 방향으로 타일을 밀어 붙이는 퍼즐 처리 흐름을 구현했습니다
- 게임 오버/일시정지/재시작 등의 상위 상태를 관리합니다
- 초기화 시 `SubSystemsManager`를 통해 전투, HP, 레벨, 강화 시스템을 참조해 전체 게임 흐름을 조율합니다

### SubSystemsManager (시스템 관리)
- `ISubSystem` 구현체를 수집해 공통 방식으로 초기화합니다
- 각 시스템은 `GetSubSystem<T>()`를 통해 참조되며, 시스템 간 직접 의존성을 줄이고 느슨한 결합을 유지하도록 설계했습니다

### Effect / ItemEffect (강화 효과 / 아이템 효과)
- `ScriptableObject` 기반 추상 구조로 설계해 데이터 중심으로 확장 및 수정이 가능하도록 구성했습니다
- 코드 변경 없이 강화 효과와 아이템 효과를 추가하거나 조정할 수 있도록 했습니다

### LevelSystem (레벨·경험치 관리)
- 레벨에 따라 몬스터 체력과 스폰 속도 배율을 조정해 난이도를 점진적으로 상승시킵니다

### UpgradeSystem (강화 선택)
- `LevelSystem`의 레벨업 이벤트를 구독해 `ScriptableObject` 기반 `Effect` 중 랜덤 2개를 선택해 UI에 표시합니다
- 선택된 강화 효과는 `PlayerStatsSystem`에 반영되며, 데이터 중심으로 강화 종류를 유연하게 확장할 수 있도록 설계했습니다

### PlayerStatsSystem (전투 스탯 집계)
- 공격력, 공격속도, 투사체 속도 등 전투 스탯을 관리하고 강화 효과가 반영된 최종 값을 계산합니다
- 스탯 관련 로직을 한 곳에 모아 전투 계산과 UI 표시의 일관성을 유지하도록 했습니다

### ProjectileSystem (자동 사격 로직)
- `PlayerStatsSystem`의 최종 스탯을 기준으로 사격을 계산합니다
- 현재 필드의 몬스터 중 가장 가까운 타겟을 찾고 `PoolingSystem`에서 투사체를 가져와 발사합니다

### MonsterSystem (웨이브/난이도 관리)
- 웨이브 단위로 몬스터를 스폰하고, 리스트로 관리합니다
- `LevelSystem`에서 제공하는 정보를 사용해 레벨이 오를수록 몬스터 체력과 스폰 간격을 조정합니다
- 웨이브 종료, 브레이크 타임, 다음 웨이브 시작 등 전투 사이클의 흐름을 관리합니다

### PoolingSystem / ObjectPool<T> (성능 최적화)
- 몬스터, 투사체, 타일 등 반복 생성되는 오브젝트를 풀링해 재사용합니다
- `Instantiate` / `Destroy` 호출을 최소화해 GC와 프레임 드랍 가능성을 줄였습니다
- 제네릭 기반 `ObjectPool<T>`로 구현해 새로운 타입도 동일한 패턴으로 확장할 수 있도록 설계했습니다

### ItemSystem (인게임 아이템 사용)
- `PlayerData.equipped` 기반으로 아이템 버튼 UI를 구성합니다
- 사용 시 수량을 소모한 뒤 `ItemEffect`를 실행해 전투, 몬스터 상태, 연출에 영향을 주도록 구성했습니다

### SaveManager (데이터 관리)
- 로컬 데이터와 서버 데이터의 흐름을 통합 관리하는 상위 매니저입니다
- `PlayerData` 변경을 감지해 Dirty 상태를 관리합니다
- 로그인 이후 데이터 로드와 서버 동기화 흐름을 담당합니다
- 서버 저장 실패 시 Pending 데이터를 유지하고 이후 재시도를 수행합니다

### SaveService (저장 처리)
- 실제 저장/로드를 담당하는 서비스 레이어입니다
- 로컬 저장(`PlayerPrefs`)과 서버 저장(`PlayFab`)을 분리해 처리합니다
- 서버 저장 시 revision 기반 비교를 통해 데이터 충돌을 방지하도록 구성했습니다

### PlayFabManager (서버 연동)
- 게스트 로그인 및 서버 통신을 담당합니다
- 로그인 상태에 따라 서버 저장 가능 여부를 판단합니다
- 서버 데이터 로드 및 저장 요청을 처리합니다

### StoreController (상점 시스템)
- 아이템 구매 UI 및 로직을 담당합니다
- 구매 시 골드를 소비하고 아이템을 `PlayerData`에 반영합니다
- 상점 종료 시 저장을 트리거합니다

### BagController (인벤토리/장착)
- 보유 아이템을 기반으로 인벤토리 UI를 구성합니다
- 아이템 장착/해제 로직을 처리하며 `PlayerData`에 반영합니다
- 장착 상태는 인게임 시스템과 연동됩니다

### UI 모듈
- `TileUI`, `HpUI`, `UpgradeUI`, `PlayerStatsUI`, `MonsterSystemUI`, `GameOverUI` 등 인게임 UI를 기능별로 분리했습니다
- 로비에서는 `GoldUI`, `ShopItemUI`, `InventorySlotUI`, `EquipSlotUI`를 통해 상점, 인벤토리, 장착 흐름을 구성했습니다
- DOTween을 활용해 타일 팝 애니메이션, 피격 연출, 강화 선택, 브레이크 타임 UI 등을 구현했습니다

## ▶ 모바일 대응
- 모바일 해상도 대응을 위해 Canvas Scaler 조정 및 Safe Area를 사용해 UI를 구성
